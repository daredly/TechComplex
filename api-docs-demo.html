API Documentation

Детальная схема API endpoints для взаимодействия между фронтендом и бэкендом

POST /api/auth/login
Аутентификация пользователя и получение JWT токена
Параметры запроса
Параметр Тип Обязательный
email string Да
password string Да
Пример ответа
Status: 200 OK Success
{ "token": "eyJhbGciOiJIUzI1NiIs...", "user": { "id": 1, "email": "user@example.com", "name": "John Doe" } }
GET /api/users/profile
Получение профиля текущего пользователя
Заголовки
Заголовок Значение
Authorization Bearer {token}
Пример ответа
Status: 200 OK Success
{ "id": 1, "email": "user@example.com", "name": "John Doe", "role": "user", "createdAt": "2024-01-01T00:00:00Z" }
PUT /api/users/profile
Обновление данных профиля пользователя
Параметры запроса
Параметр Тип Обязательный
name string Нет
email string Нет
Пример ответа
Status: 200 OK Success
{ "id": 1, "email": "new@example.com", "name": "John Smith", "role": "user", "updatedAt": "2024-01-02T00:00:00Z" }
DELETE /api/users/profile
Удаление аккаунта пользователя
Заголовки
Заголовок Значение
Authorization Bearer {token}
Пример ответа
Status: 200 OK Success
{ "message": "Account successfully deleted" }
